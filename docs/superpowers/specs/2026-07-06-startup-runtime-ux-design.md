# 启动与运行时体验打磨设计

## 目标

减少 ClearLine 测试和日常使用中的误解：启动前能明确知道为什么不能启动；运行中不能修改会影响当前音频链路的设置，避免 UI 看起来已经改变但实际管线仍使用旧配置。

## 范围

本阶段只做 UI 与状态提示打磨：

- 顶部启动按钮在缺少输入或输出设备时显示更明确的中文提示。
- 启动高质量模式时，如果用户填写了无效 DeepFilterNet 模型目录，状态信息明确说明会使用内置高质量后端。
- 运行中锁定会影响声音链路的设置：输入设备、输出设备、降噪模式、DeepFilterNet 模型目录。
- 保留已存在的运行中锁定强度和抗风噪开关行为。

## 非目标

- 不做运行中热切换设备或算法。
- 不自动停止并重启音频管线。
- 不把 DeepFilterNet 模型目录作为高质量模式的必填项；无效或空目录仍允许使用内置高质量后端。
- 不改变低延迟 RNNoise、高质量 adaptive/DeepFilterNet 的音频处理算法。

## 方案选择

推荐方案：**保守 UI 锁定 + 减少额外状态文案**。

- 使用已有 `pipeline.state().is_running()` 判断是否锁定控件。
- 设备 combo、刷新按钮、降噪模式按钮、模型目录输入框和清空按钮在运行中禁用。
- 顶部按钮文案根据可启动状态显示：
  - 运行中：`停止降噪`
  - 未运行且设备齐全：`开始降噪`
  - 未运行且缺少输入输出：`选择设备后开始`
- 启动成功后，如果 DeepFilterNet 模型目录无效但高质量模式仍启动成功，状态文案包含“模型目录无效，已使用内置高质量后端”。

备选方案 A：修改设置后自动重建管线。

- 优点：交互更直接。
- 缺点：需要处理 stream 重建、失败回滚、音频中断，风险较高。
- 当前不采用。

备选方案 B：允许运行中修改 UI，但显示“下次启动生效”。

- 优点：实现简单。
- 缺点：容易误解当前实际声音效果。
- 当前不采用。

## UI 行为

设备页：

- 运行中输入设备选择框禁用。
- 运行中输出设备选择框禁用。
- 运行中输入/输出刷新按钮禁用。
- 输出设备区域的“打开声音设置”按钮保持可用，因为它不改变 ClearLine 当前管线。

降噪模式区域：

- 运行中低延迟/高质量模式按钮禁用。
- 运行中模型目录输入框禁用。
- 运行中模型目录清空按钮禁用。
- 已有强度和抗风噪运行中锁定保持不变。

## 错误与状态提示

- 没有输入和输出设备：顶部按钮显示 `选择设备后开始`，状态沿用设备枚举提示。
- 只缺输入或只缺输出：顶部按钮显示 `选择设备后开始`，状态沿用设备枚举提示。
- 高质量模式 + 非空但无效模型目录：启动成功后状态显示 `DeepFilterNet 模型目录无效，已使用内置高质量后端：...`。

## 测试策略

自动测试覆盖：

- 顶部按钮文案在运行中、可启动、不可启动三种状态下正确。
- DeepFilterNet 启动警告只在高质量模式且非空无效路径时出现。
- 运行中不显示额外锁定提醒。
- 源码中设备/模式/模型目录控件使用 `add_enabled(!is_running, ...)` 或等效禁用路径，防止后续回归。

手动测试：

1. 打开 ClearLine。
2. 正常选择输入/输出设备并启动。
3. 运行中确认输入/输出设备、刷新、降噪模式、模型目录输入和清空按钮不可操作。
4. 运行中确认“打开声音设置”仍可点击。
5. 停止后确认上述控件恢复可操作。
6. 在高质量模式中填写无效模型目录并启动，确认状态提示使用内置高质量后端。
